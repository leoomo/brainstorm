## 1. 产品概述

### 1.1 产品定位

一款 Chrome 浏览器插件，通过让多个 AI 大模型轮流讨论，帮助用户生成高质量的产品文档。

### 1.2 核心价值

- **多角度思考**: 汇聚不同 AI 模型的视角，完善需求细节
- **高效产出**: 自动化讨论流程，快速生成规范的 Markdown 文档
- **灵活控制**: 支持多种讨论模式，用户可自定义参与模型和讨论深度

## 4. 详细功能设计

### 4.1 模型配置模块

**配置项**:
- 模型名称（自定义，如 "GPT-4"、"Claude-3"）
- API Provider（openai / anthropic / 自定义）
- API Endpoint（可选，默认使用官方端点）
- API Key

**存储方式**: chrome.storage.local

### 4.2 三种讨论模式

#### 模式 A：圆桌会议

**流程**:
```
模型A发言 → 模型B发言 → 模型C发言 → 下一轮...
```

**适用场景**: 需求细化、多角度完善

**特点**:
- 按顺序轮流发言
- 每个模型基于前一个模型的观点补充自己的看法
- 适合逐步深化需求

#### 模式 B：头脑风暴

**流程**:
```
同时要求: 模型A生成观点1
同时要求: 模型B生成观点2
同时要求: 模型C生成观点3
→ 综合汇总形成终稿
```

**适用场景**: 快速收集想法、创意发散

**特点**:
- 所有模型同时生成初稿
- 独立思考，不互相影响
- 最后由系统或指定模型汇总

#### 模式 C：辩论评审

**流程**:
```
模型A提出方案 → 模型B质疑 → 模型C补充 → 循环...
```

**适用场景**: 方案评审、风险识别

**特点**:
- 模型之间可以有不同观点
- 互相提问和质疑
- 最终综合各方意见形成文档

### 4.3 讨论控制

**轮数控制**:
- 默认: 3 轮
- 智能终止: 用户可随时点击"继续讨论"增加轮数
- 最多支持: 10 轮

**流式输出**:
- 每个模型的发言以流式方式逐步显示
- 模拟真实对话的视觉效果

### 4.4 文档生成

**输出格式**: Markdown

**文档结构**（示例）:
```markdown
# 产品需求文档

## 1. 产品概述
[综合讨论内容]

## 2. 功能需求
[详细功能列表]

## 3. 非功能需求
[性能、安全等要求]

## 4. 界面设计建议
[UI/UX 方向]

## 5. 技术建议
[技术选型建议]
```

### 4.5 历史管理

**保存内容**:
- 讨论主题/标题
- 使用的模式
- 参与的模型列表
- 讨论轮数
- 完整的消息历史
- 最终生成的文档
- 创建时间

**操作**:
- 查看历史列表
- 打开历史讨论
- 删除历史
- 导出为 .md 文件

## 5. 技术架构

### 5.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   Chrome Plugin                  │
├─────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐              │
│  │  Sidebar    │  │  Background │              │
│  │  (侧边栏)   │◄─┤   Service   │              │
│  └─────────────┘  │   Worker    │              │
│        │          └──────┬──────┘              │
│        │                 │                      │
│        ▼                 ▼                      │
│  ┌─────────────────────────────────┐            │
│  │        Storage (本地存储)         │            │
│  │  - API Keys                    │            │
│  │  - Projects & Discussions      │            │
│  └─────────────────────────────────┘            │
└─────────────────────────────────────────────────┘
```

### 5.2 数据流

```
用户输入需求
     │
     ▼
Sidebar UI 发送消息到 Background Service Worker
     │
     ▼
Service Worker 调用各模型 API
     │
     ▼
流式接收响应 → 实时推送回 Sidebar
     │
     ▼
讨论事件显示在底部控制台 (BottomPanel)
     │
     ▼
讨论结束 → 生成 Markdown 文档
     │
     ▼
保存到项目讨论列表 / 导出文件
```

### 5.3 数据结构

**API 配置**:
```json
{
  "apiConfigs": [
    {
      "id": "uuid",
      "name": "GPT-4",
      "provider": "openai",
      "apiKey": "sk-...",
      "endpoint": "https://api.openai.com/v1",
      "enabled": true
    }
  ]
}
```

**讨论历史**:
```json
{
  "history": [
    {
      "id": "uuid",
      "title": "需求主题",
      "mode": "round-table|brainstorm|debate",
      "models": ["GPT-4", "Claude"],
      "rounds": 3,
      "messages": [
        {
          "role": "assistant",
          "model": "GPT-4",
          "content": "...",
          "timestamp": "..."
        }
      ],
      "finalDoc": "# 产品文档...",
      "createdAt": "2026-03-02T21:00:00Z"
    }
  ]
}
```

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| API Key 无效 | 提示具体模型配置错误，指向配置页面 |
| 网络请求失败 | 自动重试 3 次，失败后提示用户 |
| 单个模型失败 | 跳过该模型，其他模型继续讨论，提示用户 |
| 速率限制 (429) | 排队等待 30 秒后重试，提示用户 |
| 解析响应失败 | 记录错误日志，提示用户检查网络 |

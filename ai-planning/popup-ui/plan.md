# Plan: popup-ui

## 目标

用最佳设计实践优化界面设计，提升用户体验和视觉吸引力。

## 关键决策点

### 决策 1: 设计风格定位

- **选项 A: 保持现代简洁风格** - 保持当前 #0EA5E9 主色调，微调细节
- **选项 B: 深色模式支持** - 添加深色主题切换器
- **选项 C: 全新设计系统** - 引入更现代的配色和组件风格

- **建议**: 选项 B - 添加深色模式支持，同时优化现有浅色设计
- **需要确认**: ✅ 已确认

### 决策 2: Markdown 渲染方案

- **选项 A: 纯 CSS 实现** - 使用纯 CSS 样式化 pre/code
- **选项 B: 引入 marked.js** - 引入轻量级库渲染 Markdown
- **选项 C: 使用 highlight.js** - 带代码高亮的完整方案

- **建议**: 选项 B - marked.js 轻量且足够
- **需要确认**: ✅ 已确认

### 决策 3: 流式响应显示

- **选项 A: 保持现状** - 消息一次性显示
- **选项 B: 字符级流式** - 逐字符显示 AI 回复
- **选项 C: 渐进式显示** - 逐行显示

- **建议**: 选项 B - 字符级流式显示提升用户体验
- **需要确认**: ✅ 已确认

### 决策 4: 进度指示器设计

- **选项 A: 简单加载动画** - 当前 spinner 优化
- **选项 B: 进度条 + 模型状态** - 显示每个模型的生成进度
- **选项 C: 步骤指示器** - 显示 "第 X 轮 / 最多 Y 轮"

- **建议**: 选项 B - 每个模型有独立进度显示
- **需要确认**: ✅ 已确认

### 决策 5: 组件重构程度

- **选项 A: 最小改动** - 只优化 CSS 和添加必要 JS
- **选项 B: 中等重构** - 拆分 popup.js 为模块
- **选项 C: 全面重构** - 引入 Preact/React

- **建议**: 选项 B - 中等重构，拆分 JS 模块但不使用框架
- **需要确认**: ✅ 已确认

## 实现步骤

### Phase 1: 设计系统优化
1. 更新 CSS 变量系统，增强一致性
2. 优化配色方案和阴影层次
3. 添加深色模式 CSS 变量和样式
4. 统一按钮、表单、卡片样式

### Phase 2: 核心组件改进
1. 重构 Message 组件，支持流式显示
2. 添加 Model Selector 视觉反馈（选中计数、徽章）
3. 优化 Mode Selector 交互
4. 添加 Loading/Progress 组件

### Phase 3: 视图优化
1. Main View - 优化输入区域和布局
2. Discussion View - 添加模型状态指示
3. Config View - 改进配置卡片布局
4. Document View - 集成 marked.js 渲染

### Phase 4: 交互增强
1. 添加键盘快捷键支持
2. 优化过渡动画
3. 添加 toast 通知系统
4. 改进错误处理 UI

### Phase 5: 代码重构
1. 拆分 popup.js 为模块
2. 添加状态管理抽象
3. 提取通用组件函数

## 风险评估

1. **CSS 冲突** - 需确保新样式不破坏现有功能 → 测试各视图
2. **marked.js 引入** - 需评估体积影响 → 使用 minified 版本
3. **深色模式** - 需确保所有组件都有深色样式 → 分阶段实现
4. **流式显示性能** - 频繁 DOM 操作可能影响性能 → 使用文档片段
